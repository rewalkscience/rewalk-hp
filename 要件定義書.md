## 1. プロジェクト概要

### 目的
Peatix 依存から脱却し、参加者データを自社で保有できる Web 基盤を構築する。
セミナー告知・申込・決済・フォローまでを自社サイトで完結させリピーター化を促進する。

### サイト構成
セミナー / EC サイトを親ページとし、Rewalk Science の事業紹介ページを子ページとして配置する。

| サイト | 役割 |
|--------|------|
| **セミナー / EC サイト** | セミナー親ページ・一覧・詳細・申込・決済・アーカイブ動画・会員管理 |
| **Rewalk Science 事業紹介ページ** | 事業の魅力、Evidence × Practice の考え方、提供内容、選ばれる理由を伝える LP |

## 3. 技術スタック

| レイヤー | 採用技術 | 補足 |
|----------|----------|------|
| フロントエンド | Next.js 14（App Router） | SSR / RSC でページ生成 |
| ホスティング | Cloudflare Pages + Workers | Pages でフロント、Workers で API 処理 |
| データベース | Supabase（PostgreSQL） | 認証・DB・ストレージを一元管理 |
| 認証 | Supabase Auth | メール / パスワード認証。会員権限管理に利用 |
| 決済 | Stripe Checkout | カード決済。Webhook で申込完了を DB に反映 |
| メール配信 | Resend | 自動メール（申込確認・リマインド・お礼）送信 |
| 動画 | Google Drive iframe 埋め込み | 管理画面で Drive URL を登録、FILE_ID を自動抽出してプレーヤー表示 |

## 4. データモデル（主要テーブル）

### users（Supabase Auth と連携）
| カラム | 型 | 説明 |
|--------|----|------|
| id | uuid | PK（Supabase Auth の user.id と紐付け） |
| name | text | 氏名 |
| email | text | メールアドレス |
| affiliation | text | 所属 |
| occupation | text | 職種 |
| created_at | timestamp | 登録日時 |

### seminars（セミナー）
| カラム | 型 | 説明 |
|--------|----|------|
| id | uuid | PK |
| title | text | セミナー名 |
| description | text | 詳細説明 |
| body_content | jsonb | セミナー紹介本文（見出し・太字・箇条書き・画像ブロックを含む構造化本文） |
| held_at | timestamp | 開催日時 |
| location | text | 開催場所（オンライン / 会場名） |
| price | integer | 参加費（円） |
| capacity | integer | 定員 |
| is_published | boolean | 公開 / 非公開フラグ |

### seminar_images（セミナー紹介本文内画像）
| カラム | 型 | 説明 |
|--------|----|------|
| id | uuid | PK |
| seminar_id | uuid | FK → seminars.id |
| image_url | text | 画像 URL |
| alt_text | text | 代替テキスト |
| sort_order | integer | 1〜3。本文中プレースホルダーと対応 |
| created_at | timestamp | 登録日時 |

補足:
- セミナー本文内画像は 1 セミナーにつき最大 3 枚まで。
- 管理画面の本文エディターはGUI型のリッチテキストエディターとする。
- エディターでは見出し、本文、太字、箇条書き、リンク、画像挿入をボタン操作で設定できる。
- セミナー詳細ページでは、保存された本文構造と画像ブロックの順序に沿って表示する。
- 画像のアップロード先は Supabase Storage を想定する。

## 6. 主要フロー

### セミナー申込フロー
```
ユーザー → 申込フォーム入力
→ LINE クーポンコード入力（任意）
→ サーバー側で coupons を検証し、割引後金額を算出
→ Stripe Checkout（カード決済）
→ Stripe Webhook 受信（payment_intent.succeeded）
→ enrollments.status / original_price / discount_amount / paid_amount を保存
→ coupon_redemptions を記録（クーポン利用時のみ）
→ 申込完了メール（Resend）送信
→ LINE 登録 CTA をメールに同梱
```

### LINE クーポン運用フロー
```
管理者 → 管理画面でクーポン作成（対象セミナー・割引額・期限・利用上限）
→ LINE Harness または公式 LINE でクーポンコードを配布
→ ユーザーが会員登録 / LINE 登録後、申込フォームでコード入力
→ サーバー側で有効期限・対象セミナー・利用上限・重複利用を検証
→ 割引後金額で Stripe Checkout を作成
```

補足:
- 初期運用では「LINE登録したら自動で安くなる」より、クーポンコード方式を優先する。
- LINEログイン連携なしでも開始でき、キャンペーンごとに対象・期間・割引額を変えやすい。
- 将来的に LINE Login / LIFF を導入する場合は、自動付与型クーポンへ拡張できる。
- LINE Harness を採用する場合、Rewalk 本体とは別の LINE CRM 基盤として運用し、Rewalk 側とはクーポンコード・タグ・申込完了 Webhook で連携する。
- Rewalk 本体の会員DB・決済・申込履歴は Supabase / Stripe 側を正とし、LINE Harness は配信、フォーム、ステップ配信、リッチメニュー、スコアリングの役割に寄せる。

### アーカイブ動画購入フロー
```
ユーザー → 動画詳細ページで購入ボタン
→ Stripe Checkout
→ Stripe Webhook 受信
→ video_purchases に expires_at を設定して登録
→ 購入完了メール（領収書 PDF 添付）送信
→ マイページで視聴可能に
```

### 動画視聴権限チェック
```
/mypage/videos/[id] アクセス時
→ Supabase Auth でユーザー認証
→ video_purchases を検索（user_id + video_id + expires_at > now()）
→ レコードあり → iframe で Drive 動画を表示
→ レコードなし / 期限切れ → 購入ページへリダイレクト
```

## 7. 非機能要件

| 項目 | 要件 |
|------|------|
| デザイン | 白ベース＋濃いめのブルー。シンプルで見やすく、セミナーが探しやすい構成 |
| レスポンシブ | スマホ・タブレット・PC 対応必須 |
| パフォーマンス | Cloudflare Pages による CDN 配信。LCP 2.5 秒以内を目安 |
| セキュリティ | HTTPS 必須。決済は Stripe に委託し、カード情報は自サーバーに持たない |
| アクセシビリティ | 基本的な alt テキスト・フォームラベル対応 |
